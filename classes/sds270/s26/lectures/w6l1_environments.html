Environment Basics

Similar to a named list, but

  • Unique names
  • Non-order names
  • Environment is a parent
  • No copy-on-modify behavior

Environment

library(rlang)
e1 <- env(
  city = "Northampton",
  car = "Honda",
  temp = 60,
  blueSky = T
)

Environment

e1
<environment: 0x000001ce2454f148>
env_names(e1)
[1] "blueSky" "car"     "temp"    "city"

To see what is in an environment, use env_print().

env_print(e1)

On your own: Run env_print() in the console. What does it show?

Execution Environments

  • Where the function is executed
    • Only occurs when execution happens
    • Disappears (i.e. garbage collected) once completed

Exercise

Let’s write a function that returns the environment that runs during its execution. Note that this is not the global environment.

func_env <- function() {
  x <- "what?"
  current_env()
}

env_print(func_env())
  • What is the parent environment of the execution environment shown above?

Keep these in mind!

  • Package environments (i.e. the search path) control what is available to users
  • Namespace environments control what is available to developers
