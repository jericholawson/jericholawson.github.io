From Functions to Functionals

  • Functions: to isolate actions that can be used multiple times
  • Vector: need to run action on elements in array

So? Functionals!

Functionals

  • Take a function as an input and returns a vector
    • Can run function call as we iterate through some data structure
    • Alternative to for loops
      • Cleaner code; known behavior

Functionals

Functional Types

apply()

  • base R family of functionals
    • apply(), lapply(), vapply(), tapply(), replicate()

map()

  • From purrr (which is part of tidyverse)
    • map(), map2(), pmap(), walk()

apply()

  • Returns a vector after applying a function to the margins of some array or matrix
  • apply(x, MARGIN, FUN, ...)
    • x: the array/matrix used for iteration
    • MARGIN: 1 for cycling through rows, 2 for cycling through columns
    • FUN: the function to be used on each row/column

apply()

  • General case
  • Array or matrix must be composed of same type
    • Problematic for data frames

Different Types

  • lapply(x, FUN, ...): cycles through atomic vector or list; applies function to each element and returns a list
    • x and FUN in use
  • vapply(x, FUN, FUN.VALUE, ...): cycles through atomic vector or list; applies function to each element and returns a vector
    • x and FUN in use
    • FUN.VALUE is a template for each run

Different Types

  • tapply(x, INDEX, FUN): cycles through a ragged array (i.e. with some group component); applies function to each group within array
    • x and FUN in use
    • INDEX specifies the grouping vector
  • replicate(n, expr): repeated evaluation of an expression
    • n specifies number of iterates
    • expr refers to the function call used
