Q5 from Wednesday’s Lecture

Write a for-loop on the revised if-else chain that runs through this process 20 times, storing the results for outs and runs in individual data structures.

plays <- sample(c("single", "homerun", "out"), 20, prob = c(0.25, 0.05, 0.7), replace = T)

outs = 0
onBase = 0
runs = 0
for (i in 1:length(plays)){
  if (plays[i] == "single"){
    if (onBase == 3){
      runs = runs + 1
    } else {
      onBase = onBase + 1
    }
  } else if (plays[i] == "homerun"){
    runs = onBase + runs + 1
    onBase = 0
  } else {
    outs = outs + 1
  }
}

Functions

  • Built to run specific actions on given inputs to produce outputs/outcomes
myFunct <- function(x, y, z){
  # Body
  if (x < y){
    val <- x + y/z
  } else {
    val <- y + x/z
  }
  return(val)
}

Function Components

  • Arguments (i.e. formals)
    • Inputs to the function
    • Do not need to declare types
  • Body
    • What goes on in the function
    • Have return() function
  • Environment
    • Behind the scenes
    • Determine how function finds values associated with names
# Formals

  # Body
  if (x < y){
    val <- x + y/z
  } else {
    val <- y + x/z
  }
  return(val)
}

Function Composition

  • Think of it like a math function
    • f(g(x))
  • Can nest: f(g(x))
  • Can use intermediate objects: y <- g(x); f(y)
  • Can pipe: x |> g() |> f()

Example: Function Composition

x <- 2

sqrt(exp(x)) # Nesting
[1] 2.718282
y <- exp(x); sqrt(y) # Intermediate
[1] 2.718282
x |> exp() |> sqrt() # Piping
[1] 2.718282

Lexical Scoping

  • Lexical?
    • “parse-time”, not “run-time”
  • Following four rules:
    1. Name masking
    2. Functions vs. variables
    3. A fresh start
    4. Dynamic lookup

1. Name Masking

  • Names masked until needed
    • Object names masked in function environment if defined in function
    • Function \(\rightarrow\) outer function \(\rightarrow\) package \(\rightarrow\) environment
x <- 5; y <- 6; z <- 7

myFunct2 <- function(){
  x <- 1; y <- 2; z <- 3
  return(c(x, y, z))
}

myFunct2()
[1] 1 2 3

2. Functions vs. Variables

  • Scoping applies to functions AND variables
  • Function vs. variable?
    • () \(\rightarrow\) uses function object
myFunct4 <- function(x){x + 1}

myFunct3 <- function(){
  myFunct4 <- function(x){x + 10}
  return(myFunct4(10))
}

myFunct3()
[1] 20

3. A Fresh Start

  • New call = new environment
    • Memoryless between function calls
a <- 10
myFunct5 <- function(){
  a <- a + 1
  a
}

myFunct5()
[1] 11
myFunct5()
[1] 11
  • Question: How can we update the value of a for the next function call?

4. Dynamic Lookup

  • Procrastinate and check functionality when needed
    • R looks for values when function is called, not defined
    • Function mistakes will not be apparent until you run it
a <- 15

a <- 20
myFunct5()
[1] 21

The Dots (...)

  • Passes arguments to other functions
    • Uses?
      • Nesting functions
      • Accessing methods of other parent classes
    • e.g. apply() functions
  • Useful for S3 generics
    • Need a way to allow methods to take arbitrary extra arguments

The Dots (...)

subVals <- function(y, z){
  y - z
}

addVals <- function(x, ...){
  res = subVals(...)
  return(c(x, res))
}

Exiting a Function

  • Mostly returns a value or throws an error
    • Implicit or explicit return (with return())
divides <- function(x, y){
  x / y # implicit return
}

divides <- function(x, y){
  return(x / y) # explicit return
}

Invisibility

divides <- function(x, y){
  x / y # implicit return
}
divides(8, 4)
[1] 2
divides <- function(x, y){
  invisible(x / y)
}
divides(8, 4)
print(divides(8, 4))
[1] 2
(divides(8, 4))
[1] 2
  • Use invisible() to prevent automatic printing

Errors and Exit Handlers

  • To throw an error: stop()
  • To set up an exit handler: on.exit()
    • Undoes changes done in global state
errorFunct <- function(x){
  stop("Error!")
}

errorFunct2 <- function(x){
  on.exit(cat("Goodbye!\n"), add = T)
  stop("Error!")
}

errorFunct(5)

Function Forms

  • Replacement: f(x) <- y
  • Special: if, [[, for
