Q1: Functions

We start at the function level, creating functions that we can use on the boston dataset specifically.

Important(a)

Create a function named crossSection that returns population totals for a neighborhood x across a specific indicator indicator (e.g. education, income). The function serves two purposes:

  • Provides the user with a general idea about certain cross-sections of a neighborhood based on certain categories.
  • Simplifies user input down to a neighborhood and a category.

A few things: yes, there are 306 variables. Your function only needs to have options to look at a maximum of 5 different cross-sections, with each cross-section highlighting the totality of that neighborhood’s (sub)population. For example, one cross-section might an look exclusively at females across an age range. Your function would pick the corresponding columns and return population totals for females at each age range.

Your function should employ appropriate data structures, syntax, and actions while accomplishing the task above.

Important(b)

Extending the functionality of part (a), implement an example of the following:

  • An error.
  • A warning.
  • A message.

Test the updated function on three different calls, with each one corresponding to the right condition.

Important(c)

Run your crossSection function on a neighborhood and indicator of your choice. Then, create a new function that plots an appropriate graph of the population distribution. This can use baseR or ggplot2 commands.

Important(d)

Run your crossSection function on all neighborhoods (with the same indicator) in the boston dataset using an appropriate functional.

Q2: Environments for Boston Data

We now shift to environments, creating objects within the environment itself to explore its functionality.

Important(a)

Create an environment that contains the following:

  • Object containing some metadata about the boston dataset
  • The boston dataset
  • A function that can be used on the boston dataset
Important(b)

Create a new function named mean inside your newly-created environment that prints a message that states if any NAs were removed, followed by returning the average of the total populations of all neighborhoods with base R’s mean function. No input is required.

Important(c)

Run the newly-created mean() function from your created environment.

Important(d)

Run base R’s mean() function on the total populations of all neighborhoods in the boston dataset.

Important(e)

Create another mean() function in your global environment. Then, run this mean() function on the total populations of all neighborhoods in the boston dataset.

Important(f)

Describe R’s process for picking the right mean() function when…

  • You start within the the newly created environment.
  • You start from your global environment.

Q4: Help File

Important(a)

Write documentation (in the white-space below) for one of the objects/functions you created in this HW assignment so far. Take the time to look at help files of common functions or objects (e.g. mtcars, mean(), filter()) to see the details and formatting of the help file. Your documentation should have the following:

  • Description
  • Usage
  • Arguments
  • Details
  • Value
  • Examples

Yes, we have not covered help files yet, but this will be an introduction into what to consider when making a package. You do not need to write documentation that is perfect, but it should incorporate aspects useful for someone else to reference when using your object.
