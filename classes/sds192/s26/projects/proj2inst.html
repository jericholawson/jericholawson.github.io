Background

In this project, you will be working in groups of 2-3 to incorporate various data techniques discussed in class on an analysis that you will conduct on a dataset and question of your choice. A Git workflow will be established for all members in your group, and each of you will contribute to the data cleaning, exploratory data analysis, and/or general analysis associated with your research question. The process and results will be presented in a report and another mode of your choice. By the end of this project, you will be able to:

  • Develop an appropriately-scoped and relevant research question from a dataset.
  • Apply a manageable Git workflow for all members in your team.
  • Create clear, efficient, and meaningful code to execute your analysis, including the use of functions, functionals, and tidyverse operations.
  • Examine visual aspects and metrics from your analysis, including but not limited to plots, tables, statistics, and models.
  • Summarize your process and findings in a report and creative mode of your choice.

The project will contain the following parts:

  • Proposal (5 pts; due Friday, 4/10 at 11:59pm in created repository)
  • Creative Presentation (30 pts; given Monday, 4/27 in class)
  • Report (40 pts; due Monday, 4/27 at 11:59pm in repository)
  • Repository (70 pts; due Monday, 4/27 at 11:59pm)
  • Evaluations (5 pts; due Monday, 4/27 at 11:59pm on Moodle)

Parts of several lectures will be dedicated to working on the project. Aside from these times, you will be tasked with working on this project outside of class.

Instructions

  1. Find your groupmates. You will group up in teams of 2-3 and specify this to me in your proposal. You may not work by yourself.

  2. Choose a dataset and write a central research question. Your group will find a reasonably-sized dataset based on an interest of your choice. This dataset should be legitimate with some metadata that explains how the data was collected. To that end, you can find datasets from some of these resources:

    • data.gov
    • Kaggle
    • UCI Machine Learning Repository
    • NYC Open Data

    Once your group has explored the metadata and data, you will develop a central research question that analyzes a collective group, set of variables, or some form of a relationship. From there, you can also develop smaller questions you would like to answer within an exploratory data analysis.

    The group members, data source (and a summary of its data/metadata), and a synopsis of your research direction will all be included in a proposal located in the repository your group will create in the next step.

  3. Setup a repository on GitHub and RStudio. One person will need to go to their GitHub profile. In the profile, you will create a new repository by clicking “New” at the top-right hand corner of the screen:

    From there, you will put in the following specifications for your new repository:

    Then, create the repository. You will need to invite collaborators to your repository as well. Invite all members of the group, as well as Jericho (@jericholawson) to the repository.

    Thereafter, the rest of you will create a new project in RStudio based around your new repository. To do so, go to File > New Project > Version Control > Git. Copy the new repository URL and paste it into the first field. Name your project “project2” and place the new repository into a new folder on your laptop. Then, click “Create Project”.

    Note: With everyone having access to the repository, it will be extremely imperative to pull before you commit and push. Preferably, you may want to work in a separate branch and then merge the branch at major checkpoints. Refer to lab #2 and Jericho for more information.

  4. Import data and get to know the data set. This will involve a combination of looking at the metadata and a snippet of the data itself. Consider the following questions as you examine the data:

    • Who is the creator/maintainer of the data?
    • What do the observations represent? How many variables are there?
    • How many observations and variables are in the data set?
    • What are the variables in there? What are the possible values in each variable?
    • Identify the response or variable of interest in this data set.
      • This should be numerical.
    • Are there missing values in the data set? If so, how many and is there a reason why this is the case?
      • If it appears to be random, you can omit the missing observations. If not, you will need to subset your data.

    From this point on, you will be developing 3-4 unique Quarto documents, one/two for each of the next three tasks below. Each of these tasks below will need to have use of each of the following R traits:

    • Functions
    • Use of tidyverse
    • Piping or assigning objects
    • Comments and/or code chunks

  5. Perform data tidying and possible joining. Your group will need to resolve the following issues that may show up in your data:

    • File formatting (e.g. .xlsx, shape files)
    • Missing values
    • Improper data types (e.g. non-factors, all characters)
    • Improper column or group names
    • Multiple files to merge together (either more variables or observations)
    • Pivoted in incorrect format for analysis

    Create a single Quarto document that brings in your original data files and performs data tidying, joining, and pivoting (relating to the above issues and others). You should break this code up into various chunks, labeling/commenting what each chunk does briefly in a few words.

    Some data wrangling may need to be done, particularly if you are looking at certain subsets of the data or missing data exists. Your code should properly subset the data so that you are able to complete the data visualizations.

  6. Conduct an exploratory data analysis. With a combination of code related to data wrangling and data visualization, you will explore at least three variables related (in)directly to your research direction that will give your group a better idea regarding the relationships, trends, and characteristics of your data. Unlike the first project, there are no restrictions to what variables you should explore and which tools to use – only as long as they relate to your research question.

    You can use any combination of plots (via ggplot2), summary tables (via dplyr), and metrics. If you are repeatedly using similar code elements or chunks, you are encouraged to encapsulate this code within functions and/or functionals. For your report and creative mode, you will discuss what the data indicates about the situation around your research question.

  7. Conduct an overall analysis. Naturally, this will be a continuation of your exploratory data analysis. You will now explore the variables referred to from your research question. At the bare minimum, you will present at least three summary tables, plots, and/or metrics that answer your research question. These figures should…

    • Provide objective insight into the data related to your question
    • Spark a discussion about the implications of your results

    You will further discuss these two bullet points in your report and creative mode, connecting them to an overall discussion on your research questions and its replications.

    You may be inclined to complete further analysis – predictive, cluster, causal, and others. I want to make this clear: this is not required. However, if your group feels that it will aid in a clearer and thought-provoking analysis, you are open to do so (but again, not required!).

  8. Analyze and conclude your findings. Similar to project #1, you will consider the following questions:

    • What can be said about the data? Trends? Patterns? Characteristics?
    • What does this say about the situation at hand? How does this further your understanding of the topic?
    • What are some of the challenges and pitfalls of this analysis and/or data?

  9. Write a report and create a presentation in a mode of your choice. Your group will take everything that has been discussed and create two items from your group’s research. Akin to what was seen in project #1, your group will write a 2-4 page report (double-spaced, 12 point size, a common font). More specifications can be seen in the Components section. Additionally, your group will create an item of a form of your choice, which will be presented in front of the class on the last Monday of class. More information on this can be seen in the Components section.

  10. Develop a complete repository and fill out evaluations. The final version of your repository will be shared with Jericho and will consist of code, your cleaned up data, and a README file. More specifications can be seen in the Components section.

    You will also fill out evaluations on yourself and each of your group members, describing each person’s contribution to the project. This will be done on Moodle.

Components

Proposal (5 pts; due Friday, 4/10 at 11:59pm via repository)

Your group’s proposal will contain the following:

  • Names of all group members
  • Data source
  • 2-3 sentence summary of the data and metadata
  • 2-3 sentence synopsis of your research direction, including your overall question

One person will push this to your group’s repository by the due date. Make sure to add me (@jericholawson) as a GitHub collaborator. No project late days may be used here.

This is completion-based. Your group will get feedback about the feasibility of the data and research question, with approval or revisions from my end.

Creative Presentation (30 pts; presented on Monday, 4/27 in class)

As one of the two items for presenting your research, you will be creating this item based on a mode of your choice.

The mode can be one of the following:

  • A traditional 3-4 minute presentation with slides
  • A 1-2 minute TikTok-style reel highlighting the main points
  • A blog post discussion of what your group found
  • A 2-3 minute video of your project
  • A Q&A discussing the main points of your research question

Two critical aspects will be primarily graded on:

  • (15 pts) Adherence to the stylistic aspects of the mode
    • e.g. delivery, organization, and setup in a presentation
  • (15 pts) Content of the presentation
    • Adequate information to discuss research, including background, your analysis, and implications

This piece will be presented during class on the last Monday (e.g. 4/27) of the semester. Nothing will need to be submitted. You are required to show up for the entirety of everyone’s group presentations on that day. No late work will be accepted for this part.

Report (40 pts; due Monday, 4/27 at 11:59pm via Moodle)

This report will contain all of the analysis and discussion related to the research your group is conducting with the dataset.

The report will be graded based on the following specifications:

  • (2 pts) Formatting
    • Contains 2-4 pages of text (excluding images)
    • Double-spaced, 12 point, common font (with organization)
  • (8 pts) Introduction
    • Description of dataset and topic behind research question
    • Research question(s)
  • (10 pts) Exploratory data analysis
    • Figures, tables, and/or metrics of at least four variables
    • Discussion of characteristics and metrics from above items
  • (10 pts) Main analysis
    • Figures, tables, and/or metrics centered around main variable of interest
    • Discussion of results from above items
  • (10 pts) Conclusion
    • Summary of findings
    • Examination of pitfalls in data and process

This should be in your repository by the due date. Up to two late days can be used on this part (in connection to the repository).

Repository (70 pts; due Monday, 4/27 at 11:59pm via GitHub)

The repository will contain all of the coding work completed. This includes all code, the dataset, and a README.md file (as well as the report). Your group’s repository will have the following elements:

  • (5 pts) data folder
    • The .csv file of the tidied data set.
  • (60 pts) R folder
    • Three .qmd files
      • Tidying
      • EDA
      • Overall analysis
    • Code must be working, efficient, and organized.
      • (5 pts for each file) Working: Does your code work successfully on your data?
      • (10 pts…) Efficient: Does your code use coding techniques discussed in class (e.g. functions, proper variable names, appropriate processes)?
      • (5 pts…) Organized: Is your code segmented properly and includes comments/headers briefly addressing what each part does?
  • (5 pts) README.md file
    • Title, one-paragraph description of purpose, authors, and an overview of the structure.
    • Done in a .md or .txt file.
  • (0 pts) Report (in a .pdf file)

Your group will need to share this repository to me (@jericholawson) on GitHub. This task must be done by the due date mentioned above. Two project late days may be used here if needed.

For each member in your group (including yourself), you will answer the following questions in no more than two sentences:

  • What did they contribute to the project?
  • Were there any problems working with them?
