What is tidy data?

  • Every observation has its own row
  • Every variable has its own columns
  • Every type of observation unit is a table
    • Generalizes our understanding of what can go into a cell
    • e.g. single values, list-columns, tables

Is this tidy?

        Date Nrthmptn_AQI NYC_AQI Bstn_AQI
1 03/19/2022           70      72       43
2 03/18/2022           69      60       59

Benefits of tidy data

  • Can allow for different forms through the use of pivoting
    • Particularly important when
      • summarizing cases (e.g. wider, left-to-right)
      • looking at individuals (e.g. longer, top-to-bottom)
    • e.g. blood pressure measurements at multiple times for same patient

Pivoting Longer

  • Use pivot_longer() to pivot a dataset from wider to longer format:
    • You have multiple columns of the same variable/unit you want to condense to a single column
  • Takes the following arguments:
    • cols: a series of columns to condense
      • All other columns will be kept
      • More common than not to use - to indicate which variables to keep
        • The names of those columns will become repeated rows in the pivoted data frame, and the values in those columns will be stored in a new column.
    • names_to: name of column where the column names will be stored
    • values_to: name of column where the values associated with those names will be stored
    • Various arguments to support transformations to names

Pivoting Wider

  • Use pivot_wider() to pivot a dataset from longer to wider format:
    • Need to have a column that has potential indicators for each of the rows
    • Need to have a column that has those values for each of the indicators referring to a single item of interest
  • Takes the following arguments:
    • names_from: name of column to get the new column names from
    • values_from: name of column to get the cell values from
    • Various arguments to support transformations to names

  • After

Separating Columns

  • At times, our column names or values may be named in a way that can be broken apart systematically
    • We use separate() to split values in a column into values in multiple columns:
      • e.g. place_day to place and day
  • separate() takes the following arguments:
    • col: existing column to separate
    • into: vector of names of the new columns
    • sep: a character (or more) that indicate where to separate columns
      • e.g. _ in between place and day

Example

  • Before
  • Code
  • After
  Nrthmptn_3_19 Nrthamtn_3_18  Bstn_3_19 Bstn_3_18
1            70            69 Unrecorded        59
df |> 
  pivot_longer(
    cols = everything(), 
    names_to = "Measure",
    values_to = "AQI") |>
  separate(Measure, into = c("City", "Month", "Date"), sep = "_")
# A tibble: 4 × 4
  City     Month Date  AQI       
  <chr>    <chr> <chr> <chr>     
1 Nrthmptn 3     19    70        
2 Nrthamtn 3     18    69        
3 Bstn     3     19    Unrecorded
4 Bstn     3     18    59

Breaking apart and creating list-columns

  • unnest() will break the list into an entry for each row
    • has similar functionality to pivot_longer()
  • nest() will capture all of the elements in a column belonging to shared entries and put them into a list
    • has similar functionality to pivot_wider()
  • unnest()
  • nest()
starwarsUnnested <- starwars |>
  unnest(films) |>
  select(name, hair_color, films)
