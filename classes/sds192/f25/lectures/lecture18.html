Functions

  • Statements organized to perform a specific task
    • x -> y
    • y = f(x)
    • Inputs to outputs
function_name <- function(arg1, arg2) {
  
  x <- #Some code goes here referencing arg1 and arg2
  
  return(x)
}

function_name(arg1 = input1, arg2 = input2)

Functions

  • For coding: take arguments in as inputs to perform actions that will result in an output returned to the user
function_name <- function(arg1, arg2) {
  
  x <- #Some code goes here referencing arg1 and arg2
  
  return(x)
}

function_name(arg1 = input1, arg2 = input2)
  • Inputs
  • Outputs
  • Placed within function parentheses
  • Arguments are required and should be specified in order
  • Defaults can be included
    • function(x = 5, y)
  • Typically in a return statement
    • return()
  • Can only be of one object
  • Question: What data object can be used to store objects of different types?

Why functions?

  • Removes repetition
    • Efficiency in code is important!
    • A function can house the repetitive parts – only the arguments change!
  • Reproducibility
    • Can be passed to other users, used for other datasets, etc.
  • Fewer errors
    • Less code written

Example: Calculating differences

  • Inputs, actions, and outputs
  • Function call: use the name, (), and then the inputs
my_funct <- function(arg1, arg2) {
  
  x <- arg1 - arg2
  
  return(x)
}

my_funct(7, 3)
[1] 4

Example: Calculating differences

  • Create function names that are informative, yet succinct
  • Default arguments can be included in here
    • Used to make arguments optional
  • Lists can be used to return multiple items of different types
calculate_difference <- function(arg1 = 1, arg2 = 0) {
  
  x <- arg1 - arg2
  
  return(list(result = x, args = c(arg1, arg2)))
}

calculate_difference(7, 3)
$result
[1] 4

$args
[1] 7 3
calculate_difference()
$result
[1] 1

$args
[1] 1 0
calculate_difference(arg1 = 7, arg2 = 3)
$result
[1] 4

$args
[1] 7 3

Tips

  • Have descriptive, but succinct function and argument names
  • Write functions when you are repeating code
  • Specify names to arguments when calling a function
    • Ensures you are using functionality correctly
    • Otherwise, would have to rely on ordering and default arguments

Functionals

  • R functions that use a function name as one of its inputs
    • Why? Allows for efficiency in iteration
    • Can be used on lists, arrays, and data.frames
    • e.g. map() from purrr, across(), apply()

Example of using a functional

Say I want to apply the mean function on all columns in this data frame:

BostonHousing2 |> 
  select(medv, crim, zn, indus) |>
  head(3)
  medv    crim zn indus
1 24.0 0.00632 18  2.31
2 21.6 0.02731  0  7.07
3 34.7 0.02729  0  7.07

Instead of using the mean() call four times over, we can just use a map(), across(), or apply() functional:

  • map()
  • across()
  • apply()
BostonHousing2 |> 
  select(medv, crim, zn, indus) |>
  map(mean)
$medv
[1] 22.53281

$crim
[1] 3.613524

$zn
[1] 11.36364

$indus
[1] 11.13678
BostonHousing2 |> 
  select(medv, crim, zn, indus) |>
  summarize(across(where(is.numeric), mean))
      medv     crim       zn    indus
1 22.53281 3.613524 11.36364 11.13678
BostonHousing2 |> 
  select(medv, crim, zn, indus) |>
  apply(2, mean)

22.532806  3.613524 11.363636 11.136779
