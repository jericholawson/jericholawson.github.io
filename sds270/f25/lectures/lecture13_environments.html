Quiz feedback

  • Weâ€™ve defined \(x\), \(y\), and \(z\) in our global environment
    • We also define \(x\) inside the function (which will take precedence)
    • Thus, we need to check if \(3 < y\)
x <- 10; y <- 4; z <- 8

simpleCalc <- function(x, y, z){
  x <- 3
  if (x < y){
    return(x + z)
  } else {
    return(y + z)
  }
}

Quiz feedback

  • \(3 < 4 \rightarrow\) add 3 + 8

[1] 11
  • \(3 < 4 \rightarrow\) add 3 + 8
simpleCalc(x = 7, y, z)
[1] 11

Quiz feedback

  • \(x\) is defined as 10 in our global environment

  • \(3 >= 1 \rightarrow\) add 1 + 8
  • \(x\) is 10, add to 9

Quiz feedback

  • Default method from mean() will run as a last resort
    • Defined mean() method for statsVect class
mean.statsVect <- function(x, exclude = F, trim = 0, ...){
  if (exclude == T){
    x <- ifelse(x < quantile(x, trim) | x > quantile(x, 1 - trim), NA, x)
    x <- na.omit(x)
  }
  NextMethod()
}

Environment basics

Similar to a named list, but

  • Unique names
  • Non-order names
  • Environment is a parent
  • No copy-on-modify behavior

Environment

library(rlang)
e1 <- env(
  city = "Northampton",
  car = "Honda",
  temp = 60,
  blueSky = T
)

Types of environments

  • Most environments are not created by you; theyâ€™re created by R
  • Types:
    • Global
    • Package
    • Function
    • Namespace
    • Execution

Current and global environments

  • Current: Where code is currently executing
  • Global: your workspace; where all experimenting takes place

Execution environments

  • Where the function is executed
    • Only occurs when execution happens
    • Disappears (i.e. garbage collected) once completed

Keep these in mind!

  • Package environments (i.e. the search path) control what is available to users
  • Namespace environments control what is available to developers
