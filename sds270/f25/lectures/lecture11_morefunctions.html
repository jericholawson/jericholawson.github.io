Quiz feedback: Defining classes

Two ways:

attr(mtcars, "class") = c("meanDF", class(mtcars))
class(mtcars) <- c("meanDF", class(mtcars))
  • When placed first: methods in “meanDF” will be called first

Quiz feedback: Identifying the first double vector

  • While loop or for loop with break
    • An example:
i <- 1
found <- FALSE
theMean <- 0
while(!found & i <= length(mtcars)){
  if (typeof(mtcars[, i]) == "double"){
    theMean <- mean(mtcars[, i])
    found <- TRUE
  } else {
    i = i + 1
  }
}

The dots (...)

  • Passes arguments to other functions
    • Useful when nesting functions
    • e.g. apply() functions
  • Useful for S3 generics
    • Need a way to allow methods to take arbitrary extra arguments

The dots (...)

subVals <- function(y, z){
  y - z
}

addVals <- function(x, ...){
  res = subVals(...)
  return(c(x, res))
}

addVals(x = 3, y = 2, z = 1)
[1] 3 1

Capture the dots

Exiting a function

  • Mostly returns a value or throws an error
    • Implicit or explicit return (with return())
divides <- function(x, y){
  x / y # implicit return
}

divides <- function(x, y){
  return(x / y) # implicit return
}

Invisibility

  • Use invisible() to prevent automatic printing
divides <- function(x, y){
  x / y # implicit return
}
divides(8, 4)
[1] 2
divides <- function(x, y){
  invisible(x / y)
}
divides(8, 4)
print(divides(8, 4))
[1] 2
(divides(8, 4))
[1] 2

Errors and exit handlers

  • To throw an error: stop()
  • To set up an exit handler: on.exit()
    • Undos changes done in global state
errorFunct <- function(x){
  stop("Error!")
}

errorFunct2 <- function(x){
  on.exit(cat("Goodbye!\n"), add = T)
  stop("Error!")
}

errorFunct(5)
Error in errorFunct(5): Error!
errorFunct2(5)
Error in errorFunct2(5): Error!

Function forms

  • Prefix: f(x, y)
  • Infix: x %f% y
  • Replacement: f(x) <- y
  • Special: if, [[, for
