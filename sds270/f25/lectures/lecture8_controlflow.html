Control flow

  • Can be seen in the following two ways:
    • Choices
      • if/else and switch statements
    • Loops
      • for/while loops
  • Allows us to get through one process

Situation

  • Suppose we play one half-inning of a baseball game. Keep in mind, we have the following:
    • 3 outs to end the half-inning
    • For each play, it results in a single, HR, or out (simple example)

We will use this as our motivating example for our understanding of choices and loops.

Choices

  • In R: can be seen as if/else structure, ifelse() function, or switch() function
    • Completes task based on condition
    • e.g. do this if single is hit, do this otherwise

if/else structure

if (condition){
  # do something
} else if (condition){
  # do this thing
} else {
  # do this
}
  • Works for conditions associated with singular values
    • Cannot do element-wise comparisons

Example: if/else statement

  • We can model the behavior of a single at-bat/play
if (play == "single"){
  onBase = onBase + 1
} else if (play == "homerun"){
  runs = onBase + runs + 1
  onBase = 0
} else {
  outs = outs + 1
}

ifelse()

  • Suppose I want to work with multiple logical values (i.e. element-wise comparison)
    • Use ifelse() statement
      • First argument: conditional
      • Second argument: result if element-wise condition is true
      • Third argument: result if element-wise condition is false
nums <- c(2, 5, 3, 8, 1)
ifelse(nums %% 2 == 0, nums, 0)
[1] 2 0 0 8 0

Example: if/else statement

runs <- c(0, 1, 0, 0, 0, 4, 0, 0, 0)
scored <- ifelse(runs > 0, T, F)
scored
[1] FALSE  TRUE FALSE FALSE FALSE  TRUE FALSE FALSE FALSE

Alternative: case_when()

  • From dplyr package
library(dplyr)
case_when(runs > 0 ~ "True", 
          runs == 0 ~ "False", 
          runs < 0 ~ "Runs are invalid.")
[1] "False" "True"  "False" "False" "False" "True"  "False" "False" "False"

switch()

  • Used on a character to perform certain action
stat = "mean"
switch(stat, 
       mean = mean(nums),
       median = median(nums), 
       sd = sd(nums), 
       stop("Invalid stat request."))
[1] 3.8

Loops (cont.)

  • Adding next in if statement can break iteration, break stops for loop
for (i in 1:10){
  if (i < 3){
    next
  } else if (i > 7){
    break
  }
  print(i)
}
[1] 3
[1] 4
[1] 5
[1] 6
[1] 7

For-loop notes

  • Pre-allocating the output container is recommended
    • Replacement is more efficient than adding to the output container
    • e.g. vector("list", length(x)), rep(-1, length(x))
  • Be careful about the vector you iterate over
    • : can iterate 1 up or 1 down
    • Specify which vector to iterate through

While loops and repeats

  • Two specific types of for loops:
    • while loops: repeat until condition is not satisfied
    • repeat: repeat forever until break occurs

Example: while loop

xVal = 6
while(xVal < 10){
  print(xVal)
  xVal <- xVal + 1
}
[1] 6
[1] 7
[1] 8
[1] 9

The argument against for loops

  • R succeeds when using vectorized operations
    • e.g. colSums(), adding two vectors together
    • A for-loop does so element by element, making things slow
  • If possible:
    • Use an apply() function
    • Use the map() function

apply() function

  • Functional operation, usually done on a list (or data frame)
  • apply(x, margin, function)
    • x is an array
    • margin specifies direction function will be applied over
      • 1 for function completed on each row, 2 on each column
    • function is the function being applied to object

map() function

# For loop
y <- list()
for (i in seq_along(x)) {
  y[[i]] <- my_fun(i) 
}

# Map
y <- map(x, my_fun)
