Object-oriented programming

  • Create objects of a class
    • Instance is created: consists of identity, attributes, and behaviors
  • Use of abstraction, encapsulation, and inheritance
    • Certain traits we want hidden (i.e. abstraction)
    • Packaged into an object (i.e. encapsulation)
    • Share traits with other object types (i.e. inheritance)

Real-life example

  • A car can be considered a thing…
    • And each car takes on the definition of a car
    • With its VIN, make, model, and color
    • And packaged together
    • With elements shared amongst other vehicle types
  • A car can be considered a class…
    • And each car object takes on the definition of a car class
    • With its attributes and functions
    • And encapsulated
    • With inheritance

Basic principles of OOP

  • Class: a kind of object
    • In R: numeric, factor, data.frame
  • Method: function operating on objects of specific class
    • In R: mean() for numerics or logicals
  • (Multiple) inhertiance: objects can belong to one or more classes; can share attributes
    • Parent and child classes
    • In R: tbl is a child class of data.frame

Generic functions and method dispatch

  • Generic functions have methods written for specific classes
  • When you call a generic function:
    • R uses class() to determine the class of your object
    • R looks for a method that will work for the first class
    • If it can’t find one, it will look for the second class and so forth
      • If it can’t find one, it will call the default method

# Creates generic function with rmse as name
rmse <- function(x, ...) {
  UseMethod("rmse")

OOP in Java vs. R

  • Need to specify class for object you create in Java
    • Not the case in R
# object <- constructor(args)
r <- rectangle(0, 0, 5, 5)

# generic(object, arg1, arg2)
set_size(r, 10, 15)

# what really happens:
# generic.method(object, arg1, arg2)
set_size.rectangle(r, 10, 15)
